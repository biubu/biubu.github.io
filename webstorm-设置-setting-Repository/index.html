Webstorm 设置 Setting-Repository

前端IDE
  • IDE
  • 前端
  • 使用场景: 多台电脑各自安装 IDE 设置同步的问题:
    • 一般情况下,我们用公司的开发机和自己的电脑都安装了 IDE,就前端而言, 两台电脑的 IDE 配置要保持一致,之前的做法是导出配置文件,再导入到另外的一台电脑中,好麻烦的样子. 自从 webstorm 2018 之后,官方支持了配置同步的功能,但是每次启动之前都会从服务器读取配置文件,导致每次启动都很慢.经过尝试,找到以下解决方案 ,webstorm 以及同系列的IDE 都自带setting Repository,那具体要怎么设置呢:请看一下步骤:
    • 1 首先,建立一个远程仓库 ,考虑到github访问较慢以及其他因素,国内用户建议使用 码云,创建一个空仓库;复制仓库地址 :
    • 2 打开webstorm : 依次点击: File => setting Repository,如下图所示:

  • 3 在跳出的对话框中粘贴仓库地址 ,然后点击 OverWrite Remote;等一会之后,同步到仓库之后,就说明设置成功了,这时候查看远程仓库,会看到刚刚提交的配置文件.

  • 4 然后再另外的一台电脑上打开webstorm,重复第 2 步, 到第 3 步的时候,点击OverWrite Local 就行了,这样,两台电脑的配置项就同步了,可以根据需求,在多台电脑中同步配置文件.

    注 : 以上方法适用于 jetbrains 旗下所有IDE.
